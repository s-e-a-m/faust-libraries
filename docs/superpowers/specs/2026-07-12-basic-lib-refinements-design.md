## Context

After the first publication of `seam.basic.lib` at `s-e-a-m.github.io/libraries/basic/`,
Giuseppe raised five review notes. This spec turns them into a scoped, verified plan.

Grounding facts established during brainstorming:

- `vstin` is defined only in `seam.basic.lib`; no `.dsp`/`.lib` in the whole workspace
  uses it (the only other hits are the unrelated VST3 SDK).
- The official `basics.lib` has many domain converters (`db2linear`, `semi2ratio`,
  `lin2LogGain`, â€¦) and `bpf` (piecewise-linear), but **no generic curved
  `scale(a,b,c,d)`** â€” the requested feature is a genuine gap.
- The official `basics.lib` has `selectbus`/`selectxbus` but **no list reverse/rotate**.
- Routing is conceptually its own concern: `seam.routes.lib` already exists (holds
  `matrix`/`matrixfbk`) and is currently **inactive** in `seam.lib`. Signal-bus routing
  helpers therefore do **not** belong in `basic`.
- The right-hand TOC is generated by the Minimal Mistakes theme from **all** headings
  `h1â€“h6`; the per-function `#### Usage` / `#### Test` headings flood it.
- The `libraries` collection uses `permalink: /libraries/:path/`; the page `<title>` is
  just `"basic"`, with no "faust" reference.

All new Faust definitions below were **compile-verified** with `faust 2.85.5`.

## Decisions (locked with user)

1. **Scaling:** add both `scalee` (geometric/exponential) and `scalec` (tunable power curve).
2. **List Functions:** add `lrev` (reverse a list-literal) and `lrot` (cyclic rotate),
   operating on list **values**, not signal buses. Routing helpers are out of scope
   (future dedicated pass on `seam.routes.lib`).
3. **TOC:** clamp the theme TOC to `h_max=3` (only Section `h2` + Function `h3`) **and**
   add CSS for a clearer visual hierarchy. Applies to every library page.
4. **URL:** change the collection permalink to `/faustlibraries/:path/` globally,
   no redirect (first publication, negligible breakage), and improve the page title so
   it references Faust Libraries.
5. **Remove `vstin`** and collapse the now-empty "Routing" section.

### A2. Scaling section â€” add two functions
Keep existing `scalel`. Add (with full faust2md doc blocks â€” `Usage`, `Where`, `Test`):

```faust
// geometric/exponential rescale â€” perceptually uniform (Hz, gain). Requires c,d
// same sign and non-zero.
scalee(a,b,c,d,x) = c * pow(d/c, (x-a)/(b-a));

// tunable power-curve rescale â€” curve=1 linear, >1 ease-in, <1 ease-out.
// Works for any c,d (including c=0).
scalec(a,b,c,d,curve,x) = c + pow((x-a)/(b-a), curve) * (d-c);
```

Documented constraints: `scalee` undefined for `c=0` or opposite signs (note in doc);
recommend `scalec` there. Both clamp nothing â€” caller is responsible for `x` in `[a,b]`
if a bounded output is required.

### A3. List Functions section â€” add two functions
Keep existing `revlist`. Add (with full doc blocks):

```faust
// reverse a list-literal: (10,20,30) -> (30,20,10)
lrev(l) = par(i, ba.count(l), ba.take(ba.count(l)-i, l));

// cyclic left-rotate a list-literal by k: ((1,2,3,4),1) -> (2,3,4,1)
lrot(l,k) = par(i, ba.count(l), ba.take(((i+k)%ba.count(l))+1, l));
```

`lrev` generalizes `revlist` (which only generates a descending counter) to any list.
`lrot`'s `k` is taken mod `count`, so any integer is valid; positive `k` rotates left.

  Goal: clear size/weight difference between section titles and their contents.
